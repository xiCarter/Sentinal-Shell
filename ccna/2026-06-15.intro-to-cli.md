### Concepts Locked In
- Cisco IOS CLI structure is similar to Linux —
  command-based, hierarchical, tab completion works
- Three CLI modes: User EXEC (>), Privileged EXEC (#),
  Global Configuration (config)#
- enable — moves from User EXEC to Privileged EXEC
- configure terminal — enters Global Config mode
- enable secret — encrypts password using MD5,
  significantly harder to crack
- enable password — stores password in plain text
  or weak encryption, crackable in seconds
- If both are configured, enable secret takes priority
- service password-encryption — encrypts all plain
  text passwords in the config file
- do — lets you run Privileged EXEC commands from
  inside config mode without exiting
- show running-config — displays current active config
- Cisco IOS uses ? for context-sensitive help —
  same principle as man pages in Linux
