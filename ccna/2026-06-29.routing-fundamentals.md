## CCNA Day 11 — Static IP Routes, Part 1

### Completed
- Configured static routes on a router using ip route command
- Connected multiple networks through a single router using static routing
- Verified connectivity between networks with ping

### Key Concepts
- Static routes are manually configured — the router doesn't learn them automatically
- Syntax: ip route [destination network] [subnet mask] [next hop IP]
- Without a static route, a router drops packets destined for unknown networks
- Routing is a two-way street — both directions need a route or traffic fails
