### Key Concepts
- VLSM = size each subnet to its actual host need instead of wasting a fixed block on every LAN
- Largest subnet first — allocate biggest host requirement before smaller ones to avoid overlap
- Host requirement drives prefix: 45 hosts → /26 (62 usable), 64 hosts → /25 (126 usable),
  14 hosts → /28 (30 usable), 9 hosts → /28 (14 usable)
- First usable = network + 1 (router interface), last usable = broadcast - 1 (PC)
- Point-to-point router links only need 2 usable IPs → a /30 is the efficient choice
- Static routes only needed for networks NOT directly connected to a router
- A router already knows its directly connected LANs — no static route required for those
- Routing scope reasoning: R1 connects LAN1 & LAN2, R2 connects LAN3 & LAN4
  → R1 needs static routes only to LAN3 & LAN4, R2 only to LAN1 & LAN2
- ARP handles MAC resolution within each directly connected LAN automatically
