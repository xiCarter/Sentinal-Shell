### Key Concepts
- A VLAN splits one physical switch into multiple isolated broadcast domains
- Devices in different VLANs cannot talk without a router — even on the same switch
- Access port = carries ONE VLAN, connects to an end host (PC)
- Each VLAN maps to its own subnet: VLAN10 = 10.0.0.0/26, VLAN20 = 10.0.0.64/26, VLAN30 = 10.0.0.128/26
- Gateway = each VLAN's router interface, set to last usable address here per lab spec
- Broadcast domains are contained BY the VLAN — a broadcast ping only reaches devices
  in the same VLAN/subnet, proven in Simulation Mode
- Inter-VLAN routing requires the router — traffic leaves the VLAN, hits R1, routes to the other VLAN
