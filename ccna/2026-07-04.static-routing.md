### Key Concepts
- Static routes must be configured in both directions for traffic to flow
- A missing return route causes one-way traffic — ping fails even if forward path works
- Default static route: ip route 0.0.0.0 0.0.0.0 [next-hop] — catches all unknown destinations
- show ip route shows directly connected (C) and static (S) routes
- Wrong next-hop IP = route exists but traffic goes nowhere
- Troubleshooting order: check ip route → check next-hop reachability → check both directions
- Work systematically — configure and verify one router at a time
