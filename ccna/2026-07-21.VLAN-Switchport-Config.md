## CCNA Day 17 (Lab) — VLANs & Switchport Configuration

### Completed
- Built a 6-PC topology, 2 PCs per VLAN across 3 VLANs
- Assigned IP + gateway to each PC (gateway = router's VLAN interface)
- Ran 3 physical connections from switch to router — one per VLAN
- Configured each router interface with the correct per-VLAN subnet (inter-VLAN routing)
- On the switch: set host-facing and router-facing ports to switchport mode access
- Assigned each access port to its VLAN with switchport access vlan (created VLANs in the process)
- Verified delivery in Simulation Mode via a broadcast request — watched packets flow
- Introduced to STP (Spanning Tree Protocol) during the lab

### Key Concepts
- switchport mode access = port carries ONE VLAN, for end devices
- switchport access vlan = assigns the port to a VLAN (auto-creates the VLAN if new)
- Each VLAN = its own subnet + its own router interface as the gateway
- This build uses 3 separate SW-to-router links (one per VLAN) — the pre-trunk method
  → trunking will later collapse these 3 links into 1
- Broadcast stays within its VLAN — confirmed live in Simulation Mode
- STP (Spanning Tree Protocol) prevents switching loops by blocking redundant paths
  → keeps a loop-free topology when switches have multiple interconnections
