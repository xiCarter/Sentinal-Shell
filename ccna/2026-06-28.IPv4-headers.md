## CCNA Day 10 — IPv4 Header

### Completed
- Studied the fields that make up the IPv4 packet header
- Learned the purpose of each field and how routers use them

### Key Concepts
- Version: identifies IPv4 vs IPv6 (4 bits)
- IHL: Internet Header Length — how long the header is (4 bits)
- DSCP: Differentiated Services Code Point — used for QoS, prioritizing traffic
- ECN: Explicit Congestion Notification — signals congestion without dropping packets
- Total Length: size of the entire packet including header and data
- Identification, Flags, Fragment Offset: used for fragmenting and reassembling packets across networks with different MTUs
- TTL: Time to Live — decremented by 1 at each router hop, packet dropped at 0, prevents infinite loops
- Protocol: tells the receiving host what's inside — TCP (6), UDP (17), ICMP (1)
- Header Checksum: router checks for errors in the header, drops if corrupted
- Source IP / Destination IP: where it came from, where it's going
- Options: rarely used, variable length field
